# 行为准则

Network Auth Rust 希望保持专业、友善、专注问题本身的协作氛围。参与 Issue、Pull Request、讨论或代码评审时，请遵守以下准则。

## 我们鼓励

- 尊重不同经验背景，清晰表达事实、风险和建议。
- 讨论代码、设计和验证结果，避免攻击个人。
- 在指出问题时给出可复现步骤、上下文和可行动建议。
- 接受维护者对范围、优先级和实现方式的取舍。

## 不接受

- 人身攻击、骚扰、歧视、威胁或持续挑衅。
- 泄露他人的隐私、凭据、服务器信息或未公开漏洞细节。
- 在无授权场景下要求他人执行攻击、绕过或破坏性操作。
- 重复发布无关内容、广告、刷屏或恶意噪音。

## 执行方式

维护者可以编辑、隐藏或删除违反准则的内容，也可以临时或永久限制相关账号参与项目讨论。严重安全问题会优先保护用户和部署环境，再公开技术细节。
