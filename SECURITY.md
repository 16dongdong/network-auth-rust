# 安全策略

Network Auth Rust 涉及认证、签名、加密、云存储和部署脚本。请不要在公开 Issue、Pull Request 或讨论中发布漏洞利用细节、真实配置或任何凭据。

## 支持范围

当前仓库处于公开工程案例阶段，安全修复优先覆盖：

- 后台登录、会话、签名、nonce 与权限校验。
- 客户端授权协议、加密载荷、设备签名和 token 轮换。
- 远程 API access key、secret、签名和 IP 白名单。
- 安装流程、schema 初始化、配置写入和安装锁。
- 云存储配置、上传票据、下载 token 和签名 URL。
- 发布脚本、Nginx 切换、systemd unit 和回滚流程。

## 报告方式

请通过维护者私下渠道报告安全问题，并提供：

- 受影响的模块、接口或脚本。
- 最小复现步骤。
- 可能影响范围。
- 已验证的修复建议或缓解方式。

如果需要在公开 Issue 中占位，请只描述影响范围，不附带利用代码、真实域名、IP、Token、日志或配置文件。

## 凭据处理

请不要提交或公开：

- `config/local.php`、`.env`、数据库导出或运行日志。
- 数据库密码、云厂商密钥、后台 Token、API Token、session token 或签名密钥。
- 真实域名、服务器 IP、发布路径、事故记录或用户数据。
- 上传文件、云存储对象、release 压缩包或构建产物。

如果凭据已经泄露，请立即轮换对应密钥，并从公开历史中移除相关内容。
